# Add SwiftNIO-SSL for Secure SMTP (TLS/STARTTLS)

## Summary

Replace the current raw TCP NSConnection-based `SMTPEmailSender` with a NIO channel pipeline implementation using `swift-nio-ssl`. Add STARTTLS support for port 587, with optional implicit TLS on port 465. Introduce a configurable `SMTP_TLS_MODE` env var.

## Files Changed

- `Package.swift` — add `swift-nio-ssl` dependency
- `Sources/swift-crud/Core/EmailSender.swift` — rewrite `SMTPConnection` with NIO channels; add STARTTLS flow; keep `EmailSender` protocol and `PrintEmailSender` unchanged
- `Sources/swift-crud/Core/Environment.swift` — add `smtpTLSMode` and `smtpTlsInsecure` fields
- `Sources/swift-crud/main.swift` — pass TLS options to `makeEmailSender`
- `Tests/swift-crudTests/Unit/EnvironmentTests.swift` — add tests for new env vars
- `.env.example` — document new env vars

## Detailed Steps

### 1. Package.swift — Add Dependency

Add to `dependencies`:

```swift
.package(url: "https://github.com/apple/swift-nio-ssl.git", from: "2.25.0")
```

Add to `swift-crud` target products:

```swift
.product(name: "NIOSSL", package: "swift-nio-ssl")
```

### 2. Environment.swift — Add TLS Configuration

After the existing `smtpFrom` field, add:

```swift
/// TLS mode for SMTP connections: .none, .starttls (port 587, default), or .tls (port 465).
enum SMTPTLSMode: String {
    case none
    case starttls
    case tls
}

let smtpTLSMode: SMTPTLSMode
let smtpTlsInsecure: Bool
```

Parse from env:

```swift
smtpTLSMode = mergedEnv["SMTP_TLS_MODE"].flatMap(SMTPTLSMode.init) ?? .starttls
smtpTlsInsecure = mergedEnv["SMTP_TLS_INSECURE"].map { $0 == "true" || $0 == "1" } ?? false
```

Update the test initializer to include these with defaults.

### 3. EmailSender.swift — Rewrite SMTPConnection with NIO

Replace the entire `SMTPConnection` class with a NIO-based implementation.

Key structure:

1. Use `ClientBootstrap` with the shared `NIOTSEventLoopGroup` (or create a dedicated small group)
2. Send/receive via channel pipeline
3. For STARTTLS mode (port 587):
   - Connect plain TCP
   - Read greeting
   - Send `EHLO`, read response
   - If response contains `250-STARTTLS`, send `STARTTLS`
   - Read `220 Ready for TLS`
   - Add `NIOSSLClientHandler` to pipeline with `NIOSSLContext`
   - Send `EHLO` again, then `AUTH LOGIN`
4. For TLS mode (port 465):
   - Connect with `NIOSSLClientHandler` already in pipeline (TLS from the start)
   - Read greeting, `EHLO`, then `AUTH LOGIN`
5. For `none` mode:
   - Plain TCP, legacy behavior (for dev only)

Use `NIOSSLContext(configuration:)` for the TLS context. If `smtpTlsInsecure`, configure with `trustRoots = .certificates([])` to skip certificate validation (only for local/dev testing).

Keep the existing protocol and factory signatures, just pass the new TLS options.

### 4. main.swift — Wire TLS Options

Pass the TLS mode and insecure flag to `makeEmailSender`:

```swift
let sender = makeEmailSender(from: env)
```

Update `makeEmailSender` to accept `SMTPTLSMode` and the insecure flag, and construct `SMTPEmailSender` with TLS context.

### 5. EnvironmentTests.swift — Add Tests

Add tests for `smtpTLSMode` default, override parsing, and `smtpTlsInsecure`.

### 6. .env.example — Document

Add:

```
# SMTP_TLS_MODE: none, starttls (default), or tls
# SMTP_TLS_INSECURE: true to skip certificate validation (dev only)
```

## Testing Plan

- Unit: Verify env parsing for new vars
- Integration: Mock email sender remains unchanged — no test changes needed there
- Manual: Build and run with a real SMTP server (e.g., Mailtrap, Gmail) on port 587 with STARTTLS

## Questions

- Should we use a shared EventLoopGroup with the HTTP server, or a small dedicated one for SMTP? A shared group is simpler but could affect HTTP request latency if SMTP is slow. Recommendation: small dedicated `NIOTSEventLoopGroup(loopCount: 1)` inside `SMTPEmailSender`.
- Should we expose `smtpTlsInsecure` in `.env.example` at all? Yes, but clearly marked as dev-only and not for production.
