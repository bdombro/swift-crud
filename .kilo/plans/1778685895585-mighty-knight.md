# Phase 3: Eliminate `Task {}` via NIOAsyncChannel

### Problem

Every request spawns a `Task {}` (Server.swift:170-177) ã ~2-5ôçs of overhead from task creation,
scheduler dispatch, and the promise round-trip. This is the single largest cost for `/healthz`-style
endpoints (59k req/s, vs 80k+ achievable). The architectural fix: eliminate the `ChannelInboundHandler`
+ `Task {}` pattern and move to NIO's async-native connection model (`NIOAsyncChannel`), where the
handler runs directly in structured concurrency with no per-request task spawn.

### API availability (verified against NIO 2.99.0 / NIOTS 1.18.0)

- `NIOTSListenerBootstrap.bind(host:port:)` has an async variant returning `NIOAsyncChannel<Output, Never>`
- `NIOAsyncChannel.executeThenClose` consumes inbound/outbound in a scoped async block
- `NIOAsyncChannelOutboundWriter` provides `write(contentsOf:)` for typed HTTP response parts
- `ChannelPipeline.syncOperations.configureUpgradableHTTPServerPipeline` sets up the typed HTTP pipeline
- `HTTPPart<HTTPResponseHead, ByteBuffer>` replaces `HTTPServerResponsePart` ã no manual `wrapOutboundOut` needed

### Architecture change summary

| Concern | Before | After |
|---|---|---|
| Channel binding | `NIOTSListenerBootstrap.bind(...).get()` ã `Channel` | async `NIOTSListenerBootstrap.bind(...)` ã `NIOAsyncChannel` |
| Request parsing | `NIOHTTPServerHandler` (callback on `.head`/`.body`/`.end`) | `for try await part in inbound` in async loop |
| Request assembly | Instance vars `head`, `bodyBuffer` | Local variables in loop body |
| Handler execution | `Task { try await handler(request) }` per request | `try await handler(request)` directly, no task spawn |
| Response writing | `ChannelHandlerContext.write(wrapOutboundOut(...))` | `outbound.write(contentsOf: [.head(...), .body(...), .end(nil)])` |
| Logging | `whenComplete` callback on promise | Inline in async body after handler returns |

Net effect: the `NIOHTTPServerHandler` class (94 lines) disappears entirely. The `Server` class gains
a connection handler (~50 lines). `handleRequest` + `writeResponse` + `channelRead` + state vars
are replaced by a single async loop. The `Task {}`, `makePromise`, and `whenComplete` all vanish.

```swift
// Before (lines 42-54): returns EventLoopFuture<Channel>
private func bind(host: String) async throws -> Channel {
    let bootstrap = NIOTSListenerBootstrap(group: group)
        .serverChannelOption(ChannelOptions.socketOption(.so_reuseaddr), value: 1)
        .childChannelInitializer { channel in
            channel.pipeline.configureHTTPServerPipeline(withErrorHandling: true).flatMap {
                channel.pipeline.addHandler(NIOHTTPServerHandler())
            }
        }
    let channel = try await bootstrap.bind(host: host, port: Int(self.port)).get()
    self.channel = channel
    return channel
}

// After: returns NIOAsyncChannel wrapping each connection's HTTP channel
private typealias HTTPConnection = NIOAsyncChannel<HTTPServerRequestPart, HTTPPart<HTTPResponseHead, ByteBuffer>>

private func bind(host: String) async throws -> NIOAsyncChannel<EventLoopFuture<HTTPConnection>, Never> {
    let bootstrap = NIOTSListenerBootstrap(group: group)
        .serverChannelOption(ChannelOptions.socketOption(.so_reuseaddr), value: 1)
        .childChannelInitializer { channel in
            channel.eventLoop.makeCompletedFuture {
                let config = NIOTypedHTTPServerUpgradeConfiguration<HTTPConnection>(
                    upgraders: [],  // no WebSocket or other upgrade support
                    notUpgradingCompletionHandler: { channel in
                        channel.eventLoop.makeCompletedFuture {
                            try NIOAsyncChannel<HTTPServerRequestPart, HTTPPart<HTTPResponseHead, ByteBuffer>>(
                                wrappingChannelSynchronously: channel,
                                configuration: .init(
                                    backPressureStrategy: .watermark(low: 8, high: 16),
                                    isOutboundHalfClosureEnabled: true,
                                    inboundType: HTTPServerRequestPart.self,
                                    outboundType: HTTPPart<HTTPResponseHead, ByteBuffer>.self
                                )
                            )
                        }
                    }
                )
                return try channel.pipeline.syncOperations.configureUpgradableHTTPServerPipeline(
                    configuration: .init(upgradeConfiguration: config)
                )
            }
        }
    return try await bootstrap.bind(host: host, port: Int(self.port))
}
```

**3. Update `start()` and `startAndListen()` ã consume via `NIOAsyncChannel`:**

```swift
func start() async throws {
    let channel = try await bind(host: "0.0.0.0")
    self.channel = channel.channel   // keep underlying Channel for stop()
    if let address = channel.channel.localAddress {
        boundPort = UInt16(address.port ?? Int(self.port))
    }
    try await withThrowingDiscardingTaskGroup { group in
        try await channel.executeThenClose { inbound in
            for try await connectionFuture in inbound {
                group.addTask {
                    do {
                        let httpChannel = try await connectionFuture.get()
                        try await handleHTTPConnection(httpChannel)
                    } catch {
                        // connection-level error, don't crash server
                    }
                }
            }
        }
    }
}

func startAndListen() async throws {
    let channel = try await bind(host: "127.0.0.1")
    self.channel = channel.channel
    if let address = channel.channel.localAddress {
        boundPort = UInt16(address.port ?? Int(self.port))
    }
    // Manually spin a task to consume connections; tests drive stop()
    Task {
        try await withThrowingDiscardingTaskGroup { group in
            try await channel.executeThenClose { inbound in
                for try await connectionFuture in inbound {
                    group.addTask {
                        do {
                            let httpChannel = try await connectionFuture.get()
                            try await handleHTTPConnection(httpChannel)
                        } catch { }
                    }
                }
            }
        }
    }
}
```

**Note**: `startAndListen` must return immediately so tests can proceed. Spawn a background `Task`
to consume the inbound stream. The same `Task {}` is used but only _once per server lifetime_,
not per request ã no impact on throughput.

**5. Replace `NIOHTTPServerHandler` with `handleHTTPConnection`:**

```swift
private func handleHTTPConnection(
    _ connection: HTTPConnection
) async throws {
    try await connection.executeThenClose { inbound, outbound in
        for try await part in inbound {
            switch part {
            case .head(let head):
                var body = connection.channel.allocator.buffer(capacity: 0)

                // Accumulate body across subsequent .body parts
                inner: for try await part in inbound {
                    switch part {
                    case .head: break    // ignore (should not happen within a request)
                    case .body(var buf):
                        body.writeBuffer(&buf)
                    case .end:
                        break inner
                    }
                }

                await handleRequest(
                    head: head,
                    body: body.getData(at: 0, length: body.readableBytes) ?? Data(),
                    outbound: outbound
                )
            case .body, .end:
                break  // unexpected outside .head context
            }
        }
    }
}

private func handleRequest(
    head: HTTPRequestHead,
    body: Data,
    outbound: NIOAsyncChannelOutboundWriter<HTTPPart<HTTPResponseHead, ByteBuffer>>
) async {
    let uri = head.uri
    let qIdx = uri.firstIndex(of: "?") ?? uri.endIndex
    let path = String(uri[..<qIdx])
    let query = qIdx < uri.endIndex ? String(uri[uri.index(after: qIdx)...]) : ""
    let headers: RequestHeaders = head.headers
    var request = HTTPRequest(
        method: head.method, path: path, query: query, headers: headers, body: body)
    let start = Date()

    guard let (handler, params) = routes.route(for: head.method, path: path) else {
        logRequest(start: start, method: head.method.rawValue, path: path, userId: "ANONYMOUS", statusCode: 404)
        var notFoundHeaders = NIOHTTP1.HTTPHeaders()
        notFoundHeaders.add(name: "Content-Type", value: "text/plain")
        try? await outbound.write(contentsOf: [
            .head(.init(version: head.version, status: .notFound, headers: notFoundHeaders)),
            .body(ByteBuffer(string: "Not Found")),
            .end(nil),
        ])
        return
    }

    request.routeParameters = params

    let response: HTTPResponse
    do {
        response = try await handler(request)  // NO Task {} ã direct async call
    } catch {
        var errorHeaders = NIOHTTP1.HTTPHeaders()
        errorHeaders.add(name: "Content-Type", value: "application/json")
        response = .init(
            statusCode: .internalServerError,
            headers: errorHeaders,
            body: try! HTTPResponse.encoder.encode(["message": "internal server error"]))
    }

    let userIdStr = request.wasAuthChecked
        ? request.authUserId.map(String.init) ?? "ANONYMOUS"
        : "ANONYMOUS"
    logRequest(start: start, method: head.method.rawValue, path: path,
                userId: userIdStr, statusCode: Int(response.statusCode.code))

    writeResponse(response, head: head, outbound: outbound)
}

private func writeResponse(
    _ response: HTTPResponse,
    head: HTTPRequestHead,
    outbound: NIOAsyncChannelOutboundWriter<HTTPPart<HTTPResponseHead, ByteBuffer>>
) async {
    var nioHeaders = NIOHTTP1.HTTPHeaders()
    for (name, value) in response.headers {
        nioHeaders.add(name: name, value: value)
    }
    nioHeaders.add(name: "Content-Length", value: "\(response.body.count)")
    let responseHead = HTTPResponseHead(
        version: head.version, status: response.statusCode, headers: nioHeaders)
    var bodyBuffer = ByteBuffer(bytes: response.body)
    try? await outbound.write(contentsOf: [
        .head(responseHead),
        .body(bodyBuffer),
        .end(nil),
    ])
}
```

**Note**: `writeResponse` is `async` and uses `try? await outbound.write(...)` directly ã no
`Task {}`, no promise. The `try? await` suppresses write errors gracefully. Since `handleRequest`
is already `async`, this runs inline in the connection's single async context. The `await` on
`outbound.write` is non-blocking for the NIO event loop (it suspends the Swift concurrency task
until the write completes, then resumes to process the next request on the same connection).

### Risk Assessment for Phase 3

| Concern | Severity | Mitigation |
|---|---|---|
| Typed HTTP pipeline config is unfamiliar | Medium | Pattern taken directly from NIO's `NIOWebSocketServer` example (285 lines, well-tested) |
| Async request aggregation must track state across `.head`/`.body`/`.end` | Medium | Nested `for try await inner` loop gives clean state management; `break inner` exits cleanly |
| `startAndListen` background `Task` could leak if `stop()` is called early | Low | `channel.close()` from `stop()` terminates `executeThenClose`, which cascades to cancel the task group; `deinit` shuts down event loop |
| `upgraders: []` with typed pipeline ã always triggers `notUpgradingCompletionHandler` | Low | Verified: `NIOTypedHTTPServerUpgradeConfiguration<UpgradeResult>` accepts empty `upgraders`; the `notUpgradingCompletionHandler` always fires for plain HTTP |
| `HTTPPart<HTTPResponseHead, ByteBuffer>` vs `HTTPServerResponsePart` type mismatch | Low | Structurally identical ã `.body(ByteBuffer)` replaces `.body(.byteBuffer(...))`; same `HTTPResponseHead` |
| `NIOTSListenerBootstrap: Sendable` marked unavailable (line 675) ã async bind may warn | Low | The async bind method takes `@Sendable` closures; the bootstrap is created as a local var and consumed immediately by `await`. Swift 6 may emit a warning about cross-isolation call; if so, add `nonisolated(unsafe)` annotation or use `@preconcurrency import NIOTransportServices` |
| Percent-encoding edge case (same as Phase 1 Step 2) | Low | `path.removingPercentEncoding ?? path` can be added after the `?`-split if needed |
| `NIOAsyncChannel.Configuration` default values | Low | Defaults: `backPressureStrategy: .watermark(low: 2, high: 10)`, `isOutboundHalfClosureEnabled: false`. Explicit override to `(8, 16)` / `true` shown in code. |
